scrollsolution

1.滚动条

第一种：不设定高度，自动撑开时，成为最原始的滚动条
第二种：设定高度，并且设定滚动，成为当前元素的滚动条

    height:300px;
    overflow:auto;

地址：https://github.com/zthappy/Javascript/blob/master/Scroll/demo1.html

2.背景有滚动条，同时有弹出层，禁止背景滚动

解决方案:在弹出弹框的时候将背景设置为禁止滚动，在电脑上看是没问题的，但是在手机端仍然在滚动
还要再阻止冒泡

    function openPop() {
        document.querySelector(".pop").style.display = "block";
        document.querySelector("body").style.overflow = "hidden";
    }
    function closePop() {
        document.querySelector(".pop").style.display = "none";
        document.querySelector("body").style.overflow = "";
    }
    // 对弹框浮层监听
    document.querySelector(".popup").addEventListener("touchmove",function(e){
        e.preventDefault();
    })
在微信端下拉会有黑色背景，浏览器端无 地址：https://github.com/zthappy/Javascript/blob/master/Scroll/demo2.html ## 3.弹框也有滚动，在弹框滚动时禁止背景滚动 第一种解决方案 对浮层监听touchstart事件，判断当touchstart时处于边界状态，则用js主动scroll 1px，离开边界 在第二种情况下添加代码 
var dialogContent = document.querySelector('.popcon');
dialogContent.addEventListener('touchstart', function () {
  if (dialogContent.scrollTop === 0) { // 上边界
    dialogContent.scrollTop = 1;
  } else if (dialogContent.scrollTop === dialogContent.scrollHeight - dialogContent.offsetHeight) { // 下边界
    dialogContent.scrollTop -= 1;
  }
});

地址：https://github.com/zthappy/Javascript/blob/master/Scroll/demo3.html

注意事项：
如果对ios非原生滚动添加了

-webkit-overflow-scrolling: touch;
这个属性，在ios滚动会变得很流畅，但是也会导致背景滚动的问题仍然出现，但是几率不大
